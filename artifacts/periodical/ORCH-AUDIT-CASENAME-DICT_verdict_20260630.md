## 2. 欠陥：(court, date) キーによる同日別事件の過併合

事件は (case_name, court, date) で集約されるため、**同日・同裁判所に出た別事件が1行に誤併合**される:

| canonical | 誤って混入した別事件 | 同日 |
|---|---|---|
| ハマキョウレックス事件 | 長澤運輸事件 | 平成30.6.1 |
| 新国立劇場運営財団事件 | INAXメンテナンス事件 | 平成23.4.12 |
| 三井倉庫港運事件 | 東京都と畜場/芝浦と畜場 | 平成1.12.14 |
| 物上保証人に対する不動産競売事件 | 大韓民国国籍夫婦の離婚 | 平成9.2.25 |
| 日本メール・オーダー事件 | 十和田労基署長/フットボール | 昭和59.5.29 |
| 自動車損害賠償保障法16条…訴訟（誤canonical）| 国立景観/国立マンション | 平成18.3.30 |

→ L5 で「長澤運輸事件のみ引用する評釈」が誤って ハマキョウレックス に解決される＝**主鍵の誤同定**リスク。

## 3. 注意：1,124 は上限寄り（略称ゆれと混在）

過併合疑い 1,124 のうち相当数は**同一事件の略称ゆれ**（大阪空港=大阪国際空港、石油カルテル=石油価格協定刑事、
ブルドックソース=スティール・パートナーズ＝当事者別名）で**正常**。明確な別事件併合（上表型）は数百件規模と推定。
正確な切り分けは「同日別事件」を判例DBと突合する必要があり、本検証は構造的欠陥の所在特定まで。

## 4. 推奨 v0.2（低手戻り）

1. **同日同裁判所でも別固有事件名は別 case に保つ**: variant 集約を「共通名stem を持つ表記ゆれ」に限定し、
   非重複の固有事件名（ハマキョウレックス vs 長澤運輸）は別 case_id に分割。
2. **誤 canonical の是正**: 手続名長文（自賠法16条…）が canonical になっている行を、実事件名（国立景観）へ。
3. date 無し 3,131事件（article の21.5%）は L5 解決不能＝**再現率課題**（精度ではない。判決日補完で改善可）。

→ 同日別事件を分割すれば L5 主鍵精度が大きく向上。略称ゆれ集約（正常分）は維持。

## 5. caveat（honest_report）

- head 単独ヒューリスティック。過併合 1,124 は上限（略称ゆれを含む）、明確な別事件併合は下限で数百。
- variant の大半（median=1）はクリーン。欠陥は (court,date) 共有の少数に集中。

## 6. v0.2 実装結果（owner GO 2026-06-30・worktree-casename-dict `a0fa2ca`）

過併合の確実な分割は文字列だけでは不能（当事者別名 ブルドックソース/スティールパートナーズ＝同一事件 と
別事件 ハマキョウレックス/長澤運輸 が文字列上区別不能）。→ **誤分割せず過併合をフラグ化**（AUTHOR is_ambiguous 同様）:
- `build_case_name_dict.py` に `distinct_case_names` / `is_possibly_overmerged` 列追加。
- `case_name_dict_v0.2.csv`: **1,124件（10.7%）を `is_possibly_overmerged=true`** → L5 は該当 case を単一判決へ自動解決しない。
- 確実な分割は判例DB当事者照合（外部権威・別タスク）。本v0.2は誤用防止フラグまで（リグレッション無し）。
