## 2. 設計評価（保守設計は妥当）

fuzzy マージ無効は正しい判断。日本語人名は3字前後が大半で**編集距離1の別人ペアが多発**（森本滋⇔森田章、山本和彦⇔山本龍彦）。
共著ネットワークも prolific 論者が座談会で多数と共著するためゲートが弱い。→ fuzzy 有効化は別人誤マージで article_count を汚染する。
**精度優先（誤併合を出さない）＝静的DBの著者軸の信頼性として正しい設計**。

## 3. 推奨（いずれも低手戻り）

1. **ambiguity フラグの行レベル化**: ≤2字名 809 cluster に `is_ambiguous=true` を付与し、下流が「単一人物」として扱わないようにする（精度ではなく**誤用防止**）。
2. **(任意 v0.2・recall改善)** 旧字統合は214件機能。残る取りこぼし（稀な異体字・タイプミス）は itaiji マップ拡張で recall を上げられる。ただし**精度を犠牲にしない範囲**で。
3. 本採用（著者 index v0.1 を accepted）— owner GO。

## 4. caveat（honest_report）

- ≤2字 809 のうち一部は真の単一人物（例 '岡孝'）。ambiguity は「不確実」の意であり全件誤併合ではない。
- head 単独のヒューリスティック（広域cluster は誌数で点検）。完全な人手 ground-truth ではない。
- recall（同一人物の分割）は本検証では未測定（設計が意図的に保守側＝分割許容のため）。
