# AIリーガルリサーチ 評価ルーブリック v0.1（C1〜C8 / 0–3点）

- 作成: 2026-06-16 / Claude（リモート）
- 親: `docs/2026-06-14_ai_research_evaluation_design.md`（8構成概念）。本書はその**採点表**化。
- 用途: ブラインド対比（F-A 同コーパス / F-B 現実条件）で上席が ALR/HLR ブリーフを採点。

## 0. 採点規律
- 各 C を **0–3** で採点（0=不可 / 1=不十分 / 2=実用 / 3=優秀）。証拠（該当箇所）を必ず付す。
- **門前(gate)指標 = C4・C5・C8**: いずれか **0 なら総合失格**（流暢でも採点を続けない）。幻覚した名文 < 正直で出典正しい素朴。
- 集計: 門前通過後、C1–C3・C6・C7 を主スコア、C2 を減点係数（ノイズ税）として併記。単純合計はしない（用途別重み付けは別途・評価設計C7のキャリブレーション）。

## 1. ルーブリック

### C1 網羅 / recall（gold プール比）
- 0: 主要典拠の多くを取り逃す
- 1: 主要数件のみ・重大な欠落
- 2: 大半を取得・軽微な欠落
- 3: gold プールにほぼ完全一致

### C2 精度 / precision・ノイズ税
- 0: 大半が無関係
- 1: 無関係が多く人手選別が要る
- 2: 概ね的中・少量ノイズ
- 3: 高精度・ノイズ無視可

### C3 反対線 / contrary-authority recall（★ALR差別化）
- 0: 反対説・不利判例・失効に触れない
- 1: 存在に言及するのみ
- 2: 主要な反対線を提示
- 3: 反対線＋少数説・批判・射程まで網羅

### C4【門前】接地忠実度 / grounding・幻覚
- 0: **無典拠の主張/幻覚あり**（=失格）
- 1: 一部主張が無典拠
- 2: 全主張に出典・軽微な引用ズレ
- 3: 全主張に出典＋**引用箇所が実際にそう述べていることを検証済**

### C5【門前】時間正しさ / currency
- 0: **失効・旧法を現行として使用**（=失格）
- 1: 一部が古い
- 2: 現行・軽微な日付欠落
- 3: 完全に現行＋改正/判例変更を明示

### C6 問いを超えた論点 / beyond-prompt issue-spotting（★ボトルネック破壊）
- 0: 問い以外の論点ゼロ
- 1: 些末な追加1件
- 2: 関連する新規論点1〜2件
- 3: 検証済の関連新規論点を複数

### C7 判断readiness / 人の手間
- 0: 使えず全面再作成
- 1: 大幅編集が要る
- 2: 軽編集で「自分の名で出せる」
- 3: ほぼそのまま fileable

### C8【門前】較正 / 正直さ
- 0: **過信・境界申告なし**（=失格）
- 1: 一部のみ不明を明示
- 2: 多くの不明・コーパス外を明示
- 3: known-unknown を的確に・コーパス境界/規約制限を明示

## 2. アンビエント・モード加点（評価設計§5）
固定問いでなく会議トランスクリプト入力時:
- **issue-surfacing Δ**: システムが拾い人が落とした正当論点数（事後に上席確定）。
- **lead time**: 必要時点より前に出せたか（速度でなく先回り）。
- **noise税**: 誤検知率＋棄却に要した時間。
→ Δ>0 かつ noise 許容内が「人より良い」の核。

## 3. 集計と判定
- 門前(C4/C5/C8) 全通過が前提。1つでも0 → 失格・差し戻し。
- 報告: C1–C3・C6・C7 のスコア（証拠付）＋ C2 ノイズ税 ＋ アンビエントΔ/lead time。
- ALR vs HLR は同一問題でブラインド対比し、項目別に勝敗＋差分要因を記す（どちらがAIか伏せる）。
- 自己採点バイアス回避: 自AIを甘く見ない＝お目付け役の独立監査に採点設計を載せる。

## 4. 接続（データ設計との対応）
各 C は特定データを要求（評価設計§7）: C1=TOC locator/用語グラフ, C3=引用グラフ/treatment, C4=本文OCR＋ locator, C5=法令currency, C6=論点中間層, C7=出典格付け（属性層 field_confidence）。
→ 本ルーブリックは属性観測層（採用値＋接地）と評価の橋。C4は `gate_adopted_value_grounded` の人手版。
