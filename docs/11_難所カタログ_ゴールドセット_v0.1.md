# 難所カタログ / ゴールドセット v0.1（AIフィット向け・匿名化）

- 作成日: 2026-06-07
- 用途: **ALOコネクト（AIフィットが構築）の sf_case_id 解決**の、特徴量設計・confidence較正・回帰テストに使う**ラベル付き実例集**。
- 出自: 本プロジェクトで実データ・実メールから**検証済み**の難所を匿名化して類型化。**ALOしか作れないドメイン資産**。
- データ本体: [`data/goldset_anonymized.csv`](../data/goldset_anonymized.csv)（26件、機械可読）。
- 守秘: **匿名化版**。実依頼者名・相手方名は役割記号（依頼者P/法人B/弁護士A〜C 等）。実データ接続は秘密保持＋権限境界の中で。

---

## 0. 使い方（AIフィットへ）

各エントリは「**入ってくるシグナル → 正解の紐付け先 → 根拠ルール → confidence想定 → 落とし穴**」の5点セット。

- **特徴量設計**: `link_basis`（根拠）が、resolver が持つべき特徴量・ルールの一覧になる。
- **confidence較正**: `confidence`（auto/review/unmatched）が、3段しきい値（自動確定/人レビュー/未分類）の目安。
- **回帰テスト**: 各行を `eval_set` に載せ、resolver 改修時の**デグレ検知**に使う。
- **ラベル**: `difficulty_tag` が難所タクソノミー。新しい難所が出たら追記して育てる。

> 重要原則（GPT監査でも確認済み）: これらは**確定ルールでなく強い候補特徴量**。`review` 行は自動確定せず人レビューへ。`auto` でも誤紐付けの**巻戻し（append-only link_decision_log）**を必ず持つ。

## 2. 代表エントリ（抜粋・読み物版）

### G-01 旧姓立件（離婚）★最重要
- **シグナル**: 件名「離婚の件」。依頼者は婚氏（依頼者P）、相手方は元配偶者（同姓）。
- **正解**: 受任（離婚）。ただし **SF側は旧姓（P'）で立件**されている。
- **根拠**: 離婚で旧姓に復するため旧姓で立件する実務がある。婚氏では繋がらない → **相手方氏＋分野（離婚）＋同時期**で同定、名寄せ台帳（婚氏↔旧姓）。
- **落とし穴**: 婚氏で検索して0件→未分類。**正規化では別姓で永久に繋がらない**。

### G-08 相手方代理人からのメール
- **シグナル**: 差出人が相手方の弁護士事務所ドメイン。
- **正解**: **自分（ALO）の当該受任案件**（相手方代理人とのやり取りも自案件の活動）。
- **落とし穴**: 「相手方＝別組織だから非案件」と誤判定する（一般ベンダーが最も誤る）。

### G-09 自所が手続主体（破産管財人）
- **シグナル**: 「破産管財人 ○○」宛、裁判所/債権者から。
- **正解**: 管財事件（**ALOが管財人＝手続主体**であって依頼者ではない）。
- **落とし穴**: 差出/宛先に自所名があると「内部メール」と誤判定して除外。

### G-12 利益相反確認の消去法（担当推定）
- **シグナル**: 所内 Chatter「@弁護士A @弁護士B 利益相反確認」。
- **正解**: 担当は**残りの弁護士C**（`assignee_prior`）。
- **根拠**: 利益相反確認は**担当していない弁護士に問い合わせる**運用 → 消去法。**単独で確定せず**物理フォルダ所在等で裏取り。
- **落とし穴**: @された弁護士を担当と誤認（メンション＝担当ではない）。

### G-18 SF未起票（Pattern E・規模約30%）
- **シグナル**: Boxに案件フォルダ実体あり、SFに受任レコード無し。
- **正解**: 未起票→事務局起票候補。
- **落とし穴**: SF JOINで欠落し「該当なし」で捨てる（**約30%が見えなくなる**）。

### G-26 添付↔Gmail identity bridge（既知ブロッカー）
- **シグナル**: メール添付の自動保存。`msg_key` と `gmail id` が交差しない。
- **正解**: **保留**（bridge構築まで自動保存禁止）。
- **根拠**: `DD-EMAILATTACH-001` の既知ブロッカー（identity bridge未解決、conflict_risk 1,592）。
- **落とし穴**: 添付を誤案件/重複保存。AIフィットは**この未解決前提を知っておく必要**がある。

## 3. AIフィットへの引き継ぎメモ

- この26件は**出発点**。運用で出た新難所を追記し `eval_set` を育てる前提。
- `link_basis` 列は **resolver が持つべき特徴量/ルールの逆引き**として使える。
- `confidence` 列の `auto` は少数。**大半は `review`**＝「漏れ>過剰」の方針（事務局承認を通す）。
- Box→SFの確定対応（`_alo_dir.json` / `alo_dir_index_accum.json`）と突き合わせれば、**正解ラベルの一括生成**も可能。
