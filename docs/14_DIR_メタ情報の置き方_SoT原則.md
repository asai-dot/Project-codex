# DIR / メタ情報の置き方 — 分散 × 集約 を「単一の真実(SoT)」で両立する

- 作成日: 2026-06-08
- 問い: 案件↔SFの紐付け情報（MO=matter情報: sf_record_id / matter_uri / 案件名 / 構造）は、**あちこちに分散**して持つ方がよいか、**1つに集約**する方がよいか。AIが走りやすいのはどちらか。
- 結論: **両方持つのが正解。ただし全コピーを「1つの真実(SoT)」から一方向に生成し、鮮度スタンプを付ける**。これで「あちこちに最新があってAIが見つけやすい」と「情報が錯綜しない」を両立できる。

---

## 1. 三層を全部持つ（それぞれ役割が違う）

| 層 | 実体 | 役割 / 強い場面 | AIのアクセスパターン |
|---|---|---|---|
| **分散・フォルダ内** | 各案件フォルダの `_alo_dir.json`（alo-dir-map-v2） | **locality**：今いる場所に sf_id・構造・通信方向・除外フラグがある | フォルダを歩く時に即読む（外部問い合わせ不要） |
| **分散・メタデータ** | Box フォルダ **metadata**（`sf_record_id` 等） | **検索性**：ファイルを開かずに sf_id で逆引き | `search_files_metadata` で「sf_id→フォルダ」 |
| **集約・中央索引** | DD-BOXCATALOG-001 常設索引 | **横断クエリ**：案件Xの全ファイル／弁護士Yの全件／被覆率・差分 | 1回のクエリで全体を取得 |

→ **どれも要る。** 分散はAIのlocalityに効き、集約は横断に効く。メタデータは「開かずに検索」に効く。**冗長は弱点ではなく、検索の堅牢性（retrieval robustness）になる**——*ただし下の4原則を守る限り*。

## 2. 錯綜を防ぐ4原則（これが本質）

「あちこちに最新」を**事故なく**成立させる条件は、置き方ではなく**運用規律**：

1. **SoT（真実は1つ）を決める** — フィールドごとに唯一の権威ソースを定義。
   例: `sf_record_id` の真実＝SF/案件マスタ。`case_summary` の真実＝生成元。
2. **一方向生成・二重手編集の禁止** — `マスタ → _alo_dir.json → 集約索引 / Boxメタ` の向きで**生成**する。コピーは**キャッシュ**であって独立に手編集しない。逆向きの手修正をしない。
3. **鮮度スタンプ** — 全コピーに `updated` / `snapshot_id` / `schema`。読む側は**新しい方／SoTを優先**。`_alo_dir.json` は既に `updated`・`manifest_last_scan` を持つ（この資産を活かす）。
4. **冪等リフレッシュ** — 再生成しても同じ結果。だから定期ジョブで「あちこち最新」を維持できる。**競合検出時は SoT が勝ち、reconcile ジョブが直す**。

> この4つを守れば「分散して最新があちこちにある」は**安全**。守らないと「どれが正か分からない」＝まさに錯綜になる。

## 3. AI向け「どこに何があるか」マップ（manifest of manifests）

AIが迷わないよう、**アクセスパターン→読む先**を1枚で示す（これ自体を resolver に渡す）：

| 知りたいこと | 読む先 |
|---|---|
| このフォルダの案件・構造・除外フラグ | フォルダ内 `_alo_dir.json` |
| sf_id からフォルダを引く | Box フォルダ metadata 検索 |
| 案件↔ファイルの全体・被覆・差分 | DD-BOXCATALOG-001 常設索引 |
| ファイルメタ・内容変化 | DD-BMANIFEST-001（content_sha1） |
| どれが最新か | 各コピーの `updated`/`snapshot_id` を比較、SoT優先 |

## 4. ALOへの具体推奨

1. **`_alo_dir.json` は各フォルダに置き続ける**（分散・locality）— ✅ 現状維持。AIが走りやすい。
2. **Box フォルダ metadata にも spine（`sf_record_id`/`matter_uri`）を載せる**（検索性）— 「開かずに sf_id 検索」が効く。
3. **DD-BOXCATALOG-001 を中央索引として維持**（横断・被覆・差分）。
4. **3つとも「マスタから一方向生成・タイムスタンプ付き・冪等リフレッシュ」**にする。手編集はマスタのみ。
5. **古い版（`alo_sf_box_index_v1` 64%）は廃止**して読ませない（鮮度の原則）。

## 5. 注意（やってはいけない）

- **同じフィールドを2か所で独立に手編集**（→ どちらが正か分からなくなる＝錯綜の元）。
- **タイムスタンプの無いコピー**を撒く（鮮度判定不能）。
- **生成の向きを双方向**にする（循環・不整合）。
