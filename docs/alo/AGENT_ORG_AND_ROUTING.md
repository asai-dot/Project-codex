# AIチーム 組織構造とルーティング規約（正本）

owner: asai@asai-lo.com / 最終更新: 2026-06-24

このプロジェクトの全AIが従う組織構造とルーティング規約。**全員がこれを記憶して動く。**
CLAUDE.md から参照され、各 Claude Code インスタンス起動時に読み込まれる。

## 1. 組織構造
```
                 GPT Pro（監査）  ← 全成果物を監査
                      ▲
   ┌──────────────────┴──────────────────┐
   │              ヘッド (Head)            │  設計・意思決定・ルーティング・受入検査
   │   雑誌オブジェクト等のオーナーシップ      │  （= 司令塔の Claude Code: Cloud Web / Mac Cloud Code）
   └──────────────────┬──────────────────┘
                      │ 仕事を振る
        ┌─────────────┼─────────────────────────┐
        ▼             ▼                         ▼
   ワーカーちゃん     コーデックス               ローカルちゃん
  (Worker Claude   (Codex 実行担当)        (QEN / Ollama ローカルLLM)
      Code)                                安価・大量推論。小さく切って渡す
```
- **ヘッド**: 全体設計・決定・ルーティング・受入検査・GPT Proへの監査依頼。手は動かす前に「誰に振るか」を決める。
- **ハンド（実行担当, 3種）**:
  - **ワーカーちゃん** = Worker Claude Code。汎用の重い実行（記事接合・OCR等）。
  - **コーデックス** = Codex 実行担当。
  - **ローカルちゃん** = QEN（Ollama のローカルLLM）。安価・大量・ローカル完結。**タスクは小さく切ってから渡す**。
- **監査** = GPT Pro。全ハンドの成果物を監査（`tools/gpt_audit/`）。

## 2. ルーティング規約（ユーザー指示 → 動作）
ユーザーがハンドを名指ししたら、ヘッドは**そのハンドを開いて、そのハンドに仕事を渡す**。

| ユーザー指示 | ヘッドの動作 | 起動方法 |
|---|---|---|
| 「ワーカーちゃんに仕事投げて」 | Worker を開いて仕事を渡す | `./tools/wake_worker.sh <発注書 ORCH-*.md>` |
| 「コーデックスに仕事振って」 | Codex を開いて仕事を渡す | `./tools/wake_codex.sh <発注書>`（CONFIG要確認） |
| 「ローカルちゃんに仕事振って」 | **ローカルちゃんが処理できるサイズに切り分けてから**渡す | `./tools/dispatch_local.sh <チャンク>`（CONFIG要確認） |

原則:
- ハンドへの発注書は `artifacts/periodical/ORCH-*.md`（や各レーンの発注書）に**入出力・受入基準・read-only/dry-run 指定**を明記。
- 成果物は commit→push。ファイル名は衝突回避（`P##` 系は Mac Cloud Code の誌解決専用）。
- ヘッドは成果物を**受入検査**し、必要なら **GPT Pro 監査**へ回す。
- **ローカルちゃんへは必ずチャンク分割**（コンテキスト/能力に収まる単位へ）。

## 3. 起動チャネル
- 直接起動: Mac 上で各 `tools/*.sh` を実行（ワーカーは `wake_worker.sh`）。
- 間接起動（Cloud Web→Mac）: `artifacts/periodical/.worker_trigger` に発注書パスを書いて push。
  Mac で常駐する `tools/worker_watch.sh` が拾って自動起動。→ Cloud Web からも「いつでも起こす」が可能。

## 4. 未確定（CONFIG）
- コーデックスの正確な起動コマンド（`tools/wake_codex.sh` の CONFIG 行）。
- ローカルちゃん(QEN/Ollama)のモデル名・エンドポイント（`tools/dispatch_local.sh` の CONFIG 行）。
→ owner が一度教えれば各スクリプトに焼き込み、以後は名指しで即起動。
