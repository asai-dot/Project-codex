# 設計思想ドクトリン — 手続知と「巨人の肩」の多層構造 (Living Classics)

> 本書は実装メモ（`docs/dd_procedure_design.md`）と全体地図（`docs/dd_index.md`）の**上位＝思想層**。
> ⑥手続を題材に owner（浅井先生）と往復して確定した、法情報DBの**認識論と分業の正本**。
> 個々の実装はこの思想の具現化にすぎない。迷ったらここへ戻る。
>
> 【最重要の心得】法は完全な真理ではない。我々の仕事は法を creation することでも、判断を
> 代行することでもなく、**既存の権威ある資産を整列・三点測量・差分注記して、弁護士が
> "今どの地盤に立っているか"を正直に分かる状態を作る**こと。authoring でなく alignment。

---

## 1. 中心命題：作動法 ＝ 制定法（穴あき）∪ 実務補充

実際に動いている手続（＝作動法）は、制定法だけでは閉じない。

- **法令（法律＋下位法令）＝原点・正本**。手続は精緻に書いてある。だが**書いていないことがある**：
  - **意図的空白**（委任・解釈余地。立法者がわざと埋めない）
  - **立法の欠缺（手落ち）**（不備。埋めるはずが抜けた）
- **実務本＝沈黙する法の"共著者"**。「こうすればいい」と示して欠缺を埋める。**復号器であり gap-filler**。
- ∴ **作動手続 ＝ 制定法 ∪ 実務補充**。どちらか単独では真理でない。巨人は2人いる——
  **立法者（原点・不完全）** と **欠缺を埋めた実務共同体（集合知）**。

> 戒め: AI は「法を完全な ground truth にして全部を機械的に錨づけたい」と引っ張られる。
> だが法は穴あきで、実務が構成する。この引力に毎回抗うこと。

## 2. 手続知の三層 ＋ 縦に貫く原子「要件」

```
L0 手続選択   目的/状況 →「取るか・取らないか・どれを取るか」   ← 書式本に無い・実務の本体・判断
   │ (要件で分岐・留意点・戦略)                                  主役= ②判例(射程)+③文献(解説)+①要件
L1 手続執行   選んだ手続を進める（幹＋節の条件修飾・締切・先後） ← 段取り本/書式本TOCで found・アルゴリズム
   │                                                            主役= ③段取り本 + ①条文構造
L2 書式       各局面で作る書類 ＋ その要件 ＋ 留意点            ← 書式本/リーガルで found・成果物
                                                                主役= ⑦書式 + ①要件 + ③留意点
```

- **4面（法令/文献/書式/判例）は"点"でなく"層ごと"に効く**。②判例・③解説が本当に効くのは L0（選択）。
- **要件 ＝ 縦に全層を貫く原子**。同じ要件が3役を兼ねる：L0=その手続が**使えるか**(選択ゲート) /
  L1=その局面が**有効・完了か** / L2=その書類が**法的に十分か**。出所は①法令(条文)＋③コンメンタール。
- **先後関係**も要件の一種（「Xを先にやらないとYが無効」＝順序付き要件）。
- **構造化しやすさと付加価値は逆相関**：L2書式(最も構造化可・価値低)→L1執行→**L0選択(構造化困難・価値最高)**。
  → **設計スタンス**: L0は"判断を構造化"するな（できない・危険）。**"判断の入力"を揃えて並べ、選択は弁護士に残す**。

## 3. 典拠の梯子と「地面の種別」

「引用なき主張は弾く」だけでは足りない。**欠缺補充には対応条文が存在しない**から。正しい規律は——
**地面の種別を必ず宣言し、その種別に応じた最上位の典拠を引く。**

| 種別 | 典拠（梯子の最上位へ錨づけ） | 地面 | 落とした時のリスク |
|---|---|---|---|
| 法定明文 | 条文（法律・政令・省令） | 岩盤 | 無効 |
| 法定・解釈要 | 条文 ＋ 通説/判例 | 硬 | 無効/争点 |
| 意図的空白の補充 | 通達・登記先例・審査基準（委任先の運用） | 中 | 却下 |
| 司法補充 | 判例（裁判所がどう埋めたか） | 中 | 敗訴/取消 |
| 真の欠缺補充 | 複数実務本の一致（確立実務） | 軟 | 却下・信用毀損 |
| 単一説 | 1冊の主張のみ | 最軟 | 自己責任 |

- 欠缺補充の正しさは**演繹でなく実証**（登記が通る・無効にならない＝「実務で通る」作動実績）。
- ベンコム本に **「先例索引」** が付く＝本が"どの登記先例/通達で欠缺を埋めたか"を明示＝**実務本→通達/先例への橋が既にデータにある**。欠缺補充は「典拠なし」でなく、多くが上位へ辿れる。

## 4. 三点測量 — 「法が黙る所」に集中投下

語彙でやった三点測量（JLT×学陽×有斐閣）を、手続・書式に持ち上げる。同一手続/書式を **N冊整列**：

- **一致区間 ＝ 確定**（皆こうする）。**相違区間 ＝ 裁量帯（実務の幅）**。
- **差が出るのは"法が黙る所"だけ**（法定明文では本を並べても差は出ない）。
  → **本の差分マップ ＝ 法の欠缺マップ**。
- 欠缺の上では情報量が最大：**法に無いのに全冊一致＝確立実務通説**（事実上の規範）。
  **法に無くて割れる＝未確定の争点・真の裁量**（弁護士の戦場）。

## 5. 記載事項の床 — 致命傷防止（最も実利的）

**全冊一致は偶然でない。たいてい法定記載事項**（法が「書け」と命じる要素）。落とせば無効・却下・取消＝
**実務家の致命傷**。同じ整列が**二つの商品**を同時に吐く：

- **一致区間 ＝ 法定/不可欠の床 → 抜け漏れ検出（致命傷防止）** ← 最も実利
- **相違区間 ＝ 裁量帯 → 実務の幅・交渉余地**

鍵：**条文の各号が記載事項の正準リストを既に書いている**（例 会社法199①一〜＝募集事項の法定記載事項）。
書式の各記載事項を**号にマップ**すれば表記揺れ（「払込金額」≒「出資の額」）は号IDで吸収される。

- **bottom-up（N冊の収束）⇄ top-down（条文各号）の相互照合**：一致＝堅い法定記載事項。ズレ＝要調査
  （各号にあり本が皆書かない＝条件付不要 or 手落ち／本が皆書き各号に無い＝実務必須 or 下位法令由来）。
- **「落とすな」は二色**：法定記載事項(条文あり→無効) と 実務上不可欠(条文なし→却下・恥)。条文リンクの
  有無が自動で色分け。両方致命だが機構が違う。
- 記載事項の法定性は**文脈依存**（公開/非公開・現物出資の有無で必須が変わる）＝条件付＝フローの枝に接続。

## 6. 名著蘇生 (Living Classics) — 「差分」が商品

> **本は版で陳腐化する。条文アンカーは生きる。改訂のない名著が、生き返る。**

- 自炊コーパス（asai-bookshelf）の正体＝**絶版・改訂なしの長尾の名著**。なぜ自炊したか＝商用DBに無い
  良書だから。＝**crown jewels**（蘇生の対象）。
- **朽ちるもの／朽ちないもの**を切断する：

  | 朽ちる（条文束縛の事実） | 朽ちない（payload＝蘇生の理由） |
  |---|---|
  | 条文番号・現行要件（期限・添付・公告期間）・機関の権限配分・様式細目 | 趣旨・理由づけ・実務の勘所・落とし穴・利害構造・交渉の機微・書面の論理 |

- **止揚**：陳腐化を"消す"のでなく**"明示して価値化"する**。本を殺した当の差分（法改正）が、注記されると
  蘇生の橋になる。**差分そのものが商品**。
- 機構：`名著(Y年版) ─条文アンカー→ 条文 ─[e-Govの時点別/改正履歴をdiff]→ Δ(Y→現在) → AIがΔを注記
  (注記自体を改正履歴にアンカー＝検証可能・幻覚でない) → owner/GPT 監査`。前提＝**時点別法令**（動的の lawtime が効く）。
- **差分は3方向**：①本の欠缺補充が後に立法化（本の慧眼の証明）／②本の説が判例・改正で覆った（赤信号）／
  ③本の指摘した落とし穴がまだ生きている（無注記＝今も有効）。＝名著の**格付け**にもなる。
- **AIの分**：本を書き換えない。**差分を注記するだけ**。名著の声は正本、注記は別レイヤ。

## 7. 巨人の肩・多層ドクトリン — 我々の領分は last-mile

```
法務省/e-Gov/デジタル庁  …… 法の構造・委任グラフ・法令間参照（正本）    ← 主権者がやる
  ▲
商用DB(bencom/legallib)   …… 書式・実務書の綺麗なTOC                    ← 出版/DB事業者がやる
  ▲
実務本の著者              …… 欠缺の補充・勘所（名著）                    ← 専門家がやった
  ▲
【我々】                  …… 整列・三点測量・差分注記・名著蘇生・抜け漏れ検出 ← last-mile だけ
```

- **汎用なものは全部、上の誰かがやる。我々がやるのは"この事務所の蔵書×この実務"に固有の last-mile のみ。**
  芸は**線の引き方**にある。
- **委任事項（条文→政令→省令の委任グラフ）は主権者の仕事＝やらない・追随する**。理由：(a)主権スケール、
  (b)自前は導出物・国のは正本（レースは無駄かつ勝っても負け）。
- **委任は"解決"せず"記録"する（穴を一級市民に）**：
  ```
  根拠 = { 条: 商業登記法XX, 委任: true, 委任文言: "法務省令で定める",
           委任先: null(未解決), 解決源: "e-Gov/法務省 法令データ(将来)" }
  ```
  穴は失敗でなく**上位の巨人への遅延依存**。e-Gov が委任リンクを出したら `委任先` を埋める＝前方互換。
  委任文言の検出は条文テキストから機械的に可能＝**「委任がある」事実は今すぐ記録、解決は国**。
- **「追随」は「停止」でない**。3つの安全弁：(1)穴があっても動く(degraded-functional)、(2)待たない
  (見える委任文言＋現行条文で暫定錨)、(3)来たら即 consume(穴の形を標準に合わせる)。

## 8. 真偽でなく「校正された足場」— ソフト地盤の成果物

- 硬い地盤（法定明文）＝条文が足場＝**演繹的な真偽**。
- 軟らかい地盤（欠缺・解釈）＝真偽はつかない。要るのは**地に足がつくこと＝拠り所**。
  「**書いてあった！よかった**」の安堵＝**リスクの移転/分有**（公刊見解に従った＝善管注意義務の証）。
  **何もないところから自分で解釈を立てるのは裸＝危険**。だから実務家は本を**買い漁る＝足場の被覆**。
- **止揚（真偽 vs 足場）**：同じシステムが地盤で別の商品を出し、**"今どっちの地盤か"を正直に告げる**。
  種別はメタデータでなく「あなたは**正しい**／あなたは**カバーされている**」の差（身構えが違う）。
- **システムの再定義＝蔵書は"足場検索エンジン"**。L0の最重要入力＝**拠り所そのもの**（「Xと論じたい→誰が・
  どこで・何と書いたか」を最大 recall で surface）。＝自炊長尾の取り込みの正当化（真実でなく**足場の被覆**）。
- **反（偽の足場は踏み抜く）**：「書いてある」の安堵は罠にもなる（悪い本・少数説・**判例で否定**）。だから
  足場検索は **荷重格付け（load rating）** を付ける——**三点測量がそのまま荷重計**：

  | 足場の格 | 耐荷重 |
  |---|---|
  | N冊一致＋通説 | 岩盤近い |
  | 有力説（複数だが争い） | 厚い板 |
  | 単一説（1冊のみ） | 細板（要注意） |
  | 判例で否定/旧版 | **腐った板（踏むな）** |

  **安堵は与える、ただし校正して与える**。

## 9. 止揚の系譜（誤りの訂正記録）— 未来のエージェントへの申し送り

このドクトリンは AI の素朴な誤りを owner が毎回正して得られた。**同じ轍を踏まないための負の記録**：

| 私(AI)の素朴な正 | owner の反 | 合（採用） |
|---|---|---|
| 手続＝一軸のソケットに刺さる分類タグ | 手続は複雑に分岐するチャート | 手続＝**ほぼ線形の幹＋節での条件修飾**（自由DAGでもタグでもない） |
| 4面を1点に統合する（build） | 実務書が既に4面を織ってある | 統合は **found**（本にある）。我々は再リンクするだけ |
| spine を上から24類型 author | 本の業務一覧が実在の粒度 | spine は **bottom-up**（本から）。私の24類型は監査用の骨 |
| owner が手続フローを手作り | ポリシー違反（巨人の肩） | フローは**既存資産から抽出**、owner は**監査** |
| 法令＝完全な ground truth | 法は穴あき（意図的空白＋欠缺） | 作動法 ＝ **制定法 ∪ 実務補充** |
| 条文出典を全 node 必須 | 欠缺補充に条文は無い | **地面の種別を宣言**し種別相応の典拠（梯子） |
| ゴールは真偽・確定 | 軟地盤に真偽は無い、要るのは足場 | **真偽 と 校正された足場**を地盤で出し分ける |

> 教訓：AI は「演繹で閉じたい・法を完全にしたい・自分で整えたい」へ傾く。実務は実証・穴あき・
> 巨人依存で動く。**疑ったら "これは authoring になっていないか / 上の巨人がやる仕事ではないか" を問う。**

## 10. 我々がやること / やらないこと（境界の確定）

**やる（last-mile・固有）**：
- 既存資産の**整列**（本の経路を条文構造へ／N冊を局面・記載事項で）
- **三点測量**（一致＝床/確定、相違＝裁量/争点、荷重格付け）
- **差分注記**（名著×現行法Δ／本の予言の当否）
- **地面の種別宣言**と種別相応の典拠錨づけ
- **抜け漏れ検出**（法定/実務必須の記載事項の床）
- **足場検索**（拠り所の最大 recall ＋ 校正）

**やらない（巨人がやる・主権者領域）**：
- 法令の構造化・委任グラフ・法令間参照（→ 法務省/e-Gov に追随）
- 手続フロー・解釈の **創作**（→ 実務本から抽出、判断は弁護士）
- 商用DBが持つ綺麗なTOC・書式の再生産（→ bencom/legallib を consume）

## 11. 残された反（オープンな限界・next の種）

1. **接ぎ木でなく枯死**：改正で前提が根本から変わった本は蘇生不能。蘇生可否の判定が要る。
2. **上流の仕様変更リスク**：追随する以上、e-Gov 標準変更・bencom 廃業に**追随し続けるコスト**を背負う。
3. **過依存リスク**：足場を出しすぎると"カンニング"化し弁護士の判断力が痩せる。拠り所は
   **松葉杖か義足か車椅子か**——どこまで支えどこから歩かせるかの設計問題。
4. **集団的誤り**：全冊一致が"皆で同じ間違い"の可能性。条文照合が防波堤だが、条文の無い欠缺では弱い。
5. **記載事項の意味的正規化**：表記揺れ→号IDの突合は難物（②判例・条文文言を正準IDに使う）。
