## 4. レビュー観点（本体プロジェクトの評価基準を踏襲）

本体は GPT-5 レビューで以下を是正済み。同じ基準で本文書群も評価してほしい：

- [ ] **過剰主張の検出**: 「完全/必ず/最適/正解」等の断定。証拠に比して強すぎる表現を**弱化提案**。
- [ ] **確定/仮説/検証要のタグ妥当性**: 「実証済み」と書いた主張は本当に実データ根拠があるか。
- [ ] **3値整合**（採用/条件付き採用/要検証）で各設計判断を評価。
- [ ] **反例・破綻ケース**: 規模拡大（弁護士増）、名称大量不一致、チャネル増加で壊れないか。
- [ ] **本体との非重複・整合**: 本体ルールv1.1/データモデルと矛盾しないか。重複して混乱を生まないか。
- [ ] **久保さん（実装者）視点の充足**: これで実装に入れるか。**最初に決めるべき設計判断の抜け**は？
- [ ] **失敗時設計**（誤紐付けの検知・巻き戻し、API障害、認証切れ、bot暴走）への言及は足りるか。
