## 1. 入力シグナルの棚卸し（やること1）

判別に使えるシグナルを源別に列挙し、各シグナルの**判別力**（個別相談 vs 非相談を分ける力）と**信頼度**（自然発生データとしての安定性）を評価する。出典は W-190/W-210。

### 1.1 シグナル一覧と判別力

| # | シグナル | 取得源 | 観測内容（W-190/W-210根拠） | 判別力 | 信頼度 | 効く方向 |

| **SIG-2** | availability=FREE かつ transparency=transparent | Calendar event | 型D（事務所会議の議題行）は同時刻ブロック群が FREE/transparent で並ぶ＝実会議でなく**案件ステータス棚卸し掲示**（W-190 §1.4） | **高（負シグナル）** | 高（構造値・人手揺れ小） | 両立→型D会議議題として除外を強く支持 |

### 1.2 判別力の要約

- **正シグナル（個別相談を支持）の主役 = SIG-1（事件メモDoc添付）**。源内 ID完全一致で最も信頼でき、KPI分母（個別相談件数）の主判定軸。
- **負シグナル（非相談を支持）の主役 = SIG-2（FREE/transparent）と SIG-3/SIG-4（公益語彙・ML差出人）**。型D（会議議題）と型E/Gmailノイズを高判別力で落とせる。
- **語彙単独（SIG-7）は不可**: `相談`語は個別・公益・会議議題に共通出現（W-190 §1.3）。語彙は他シグナルとの**複合**でのみ用いる。
- **参加者（SIG-6）は同定キーとして非信頼**（W-190 §1.4）。除外の主軸にはしない（外部団体名の補強のみ）。

## 2. 判別アルゴリズム（やること2）

源（Calendar / Gmail / Doc）ごとに前処理が異なるため、**源で分岐 → 各源で決定木 → 共通スコアリングで灰色帯を補完**する2段構成とする。型A-E（W-190）と Gmailノイズ（W-210）を全て正しく落とせることを設計要件とする。

### 2.1 出力クラス（4分類）

| クラス | 意味 | KPI分母への扱い |
|---|---|---|
| `INDIVIDUAL`（個別相談） | 個別案件の相談（型A/B/C） | **分母に算入**（consultation_record採用候補） |
| `EXCLUDE_MEETING`（型D会議議題） | 事務所会議の案件ステータス掲示行 | 分母から除外 |
| `EXCLUDE_PUBLIC`（型E公益/勉強会/無料相談） | 非個別案件 | 分母から除外 |
| `EXCLUDE_NOISE`（MLノイズ） | 京都弁護士会ML等の非個別メール（Gmail限定） | 分母から除外 |

### 2.2 決定木（源別・優先順位＝上から評価し最初に該当した枝で確定）

**負シグナルを先に評価して除外を確定させ、残りを正シグナルで個別相談判定する**（型混在を確実に落とすため除外優先）。

#### (A) Calendar 予定 の決定木

```
D1: SIG-4 ML由来（差出人/list-id がML）          → 該当しない（Calendarに差出人概念なし。Gmail木で評価）
D2: SIG-3 公益/外部団体 語彙にマッチ
      （当番弁護士|区役所|商工会議所|協議会|勉強会|なんでも相談会|無料相談|センター電話）
        → EXCLUDE_PUBLIC（型E）で確定
D3: SIG-2 availability=FREE AND transparency=transparent
      （任意で SIG-5 `[事務所会議...]`プレフィクス / `＿<分野>`語 で補強）
        → EXCLUDE_MEETING（型D）で確定
D4: SIG-1 事件メモDoc添付あり（fileUrlにdoc_id）
        → INDIVIDUAL（型A/B/C・添付あり）で確定（confidence高）
D5: 添付なし・上記いずれも非該当
        → §2.3 スコアリングへ（灰色帯：型A-C添付なし救済 / 取りこぼし型D-E検査）
```

#### (B) Gmail スレッド の決定木

```
G1: SIG-4 差出人/list-id がML（京都弁護士会 Katal 等）
        → EXCLUDE_NOISE（MLノイズ）で確定 ★W-210の最大ノイズ源を最優先で除去
G2: SIG-3 件名が公益/外部団体 語彙（当番|区役所|交替依頼|勉強会|無料相談 等）
        → EXCLUDE_PUBLIC（型E）で確定
G3: 件名が個別案件問合せ様式（個人名＋相談/受任通知/見積 等）かつ非ML
        → INDIVIDUAL候補（ただし相談eventとの確定突合は背骨ID不在で不可・候補どまり, W-210 §4）
G4: 上記いずれも非該当
        → §2.3 スコアリングへ（灰色帯）
```

#### (C) Doc（事件メモ） の判別

Doc単独では分類しない（本文は秘匿特権で非取り込み, W-190）。Doc は **SIG-1 経由で Calendar event に従属**する。`event_id↔doc_id`（添付fileUrl）で結合した相談eventのクラスを Doc が継承する。様式M/G（手打ち/Geminiメモ）の別はクラス判定に使わない（命名fuzzy救済の補助のみ）。

### 2.3 スコアリング（決定木の灰色帯のみに適用）

決定木で確定しなかった予定/スレッド（D5/G4）に、重み付きスコアで暫定クラスと confidence を与える。**スコアは確定でなく確信度の算出**であり、§3 の HITL 境界に従って自動確定/人手確認/自動除外に振り分ける。

| シグナル | 個別相談スコア(+) | 非相談スコア(+) | 重み根拠 |
|---|---:|---:|---|
| SIG-1 事件メモDoc添付あり | +5 | — | 主シグナル・ID完全一致（W-210 添付有り率71%） |
| SIG-2 FREE+transparent | — | +5（型D） | 構造値・型D高判別 |
| SIG-3 公益/外部団体語彙 | — | +5（型E） | 定型語彙・型E高判別 |
| SIG-4 ML差出人 | — | +6（ノイズ） | W-210最大ノイズ・送信元安定 |
| SIG-5 `[事務所会議]`/分野語 | — | +2（型D補強） | SIG-2と共起する補強 |
| SIG-6 外部団体attendees | — | +1（型E補強） | 任意・非信頼 |
| SIG-7 当事者名＋相談様式（型A-C） | +2 | — | 弱い正・語彙単独不可 |
| SIG-8 進捗棚卸し語 | — | +1（型D補強） | 自由記述・補助 |
| SIG-9 Notta共有URL | +1 | — | 疎・弱い正 |

`score_individual = Σ(個別+)`、`score_exclude = Σ(非相談+)`。判定は両スコアの差と絶対値で §3 の帯に割り当てる。

### 2.4 型A-E ＋ Gmailノイズ の網羅対応表

| W-190/W-210 区分 | 本ルールでの捕捉枝 | 出力クラス | 落とせるか |
|---|---|---|---|
| 型A 新規相談（来所/web） | D4（添付あり）/ D5→SIG-7+SIG-1 | INDIVIDUAL | 採用（分母算入） |
| 型B 個別相談 | D4 / D5→SIG-7 | INDIVIDUAL | 採用 |
| 型C 法律相談（分野付き・個別） | D4 / D5→SIG-7（公益語彙無を確認） | INDIVIDUAL | 採用 |
| 型D 事務所会議の議題行 | **D3（SIG-2）**＋SIG-5/SIG-8補強 | EXCLUDE_MEETING | **除外（落とせる）** |
| 型E 公益/勉強会/無料相談 | **D2/G2（SIG-3）**＋SIG-6補強 | EXCLUDE_PUBLIC | **除外（落とせる）** |
| Gmail 京都弁護士会MLノイズ | **G1（SIG-4）** 最優先 | EXCLUDE_NOISE | **除外（落とせる）** |

→ 型A-C は INDIVIDUAL、型D/E と Gmailノイズは各除外クラスへ確定的に分離でき、**全区分を網羅**する。

## 3. 確信度と HITL 境界（やること3）

AI推定は候補どまり・人間承認で確定（**G-19**）。本ルールの自動分類は、KPI分母の機械集計に使う**自動帯**と、人間に回す**灰色帯**を確信度で分ける。自動帯であっても「確定（human_decision）」へは昇格させず、最終的な相談記録の確定（converted等の重大遷移）は別途 HITL（W-200 受入基準2/5）に従う。

### 3.1 確信度区分と振り分け

| 帯 | 条件 | 自動処理 | HITL |
|---|---|---|---|
| **自動INDIVIDUAL（高確信）** | 決定木 D4 で確定（SIG-1 添付あり・公益語彙/FREE&transparent 非該当） | 個別相談として**分母に自動算入（候補）** | 不要（ただしKPI確定はW-200受入基準に従う） |
| **自動EXCLUDE（高確信）** | 決定木 D2/D3/G1/G2 で確定（負シグナル明確） | **分母から自動除外** | 不要 |
| **灰色帯（要人手確認）** | D5/G4 のスコアリング行き、または `\|score_individual − score_exclude\| < 3`、または添付なしINDIVIDUAL候補（型A-C救済）、または Gmail INDIVIDUAL候補（背骨ID不在で event突合不可・W-210 §4） | `review.status=pending` で**人手確認キューへ**。AI推定値は候補として併記 | **必要**（人間が個別/非相談を承認） |

### 3.2 HITL 境界の具体（owner裁定 vs worker推論の分離）

- **owner裁定（G-19）**: AI推定は人間承認で確定。本ルールの全出力は「AI/自動分類=候補」であり、承認されるまで確定相談件数に算入しない（厳守）。
- **[worker推論] 灰色帯の閾値**: `差<3` および「添付なしINDIVIDUAL候補」「Gmail個別候補」を灰色帯とする閾値・スコア重みは **owner未裁定**。承認で確定化すべき候補（W-200 §7 worker推論一覧に追加）。
- **確定にしない境界（forbidden 厳守）**: スコアが高くても自動帯の個別判定を `human_decision` 扱いにしない。受任・コンフリ等の重大遷移は HG-01/HG-02 と W-200 受入基準に従う（本ルールは相談/非相談の前段フィルタに限定）。

## 4. 適用先と KPI分母への効き方（やること4）

### 4.1 源別の適用方法

| 源 | 入力 | 適用 | 出力 |
|---|---|---|---|
| **Calendar 予定** | `asai@` 等の `相談`ヒット予定 | §2.2(A) 決定木 → 灰色は §2.3 | クラス＋confidence。INDIVIDUAL=consultation_key（event_id）採用候補 |
| **Gmail スレッド** | `subject:(相談)` 等のスレッド | §2.2(B) 決定木 → 灰色は §2.3。**G1のML除去を最優先** | クラス＋confidence。INDIVIDUAL候補は event突合不可のため候補どまり（W-210 §4） |
| **Doc 事件メモ** | event添付/命名fuzzy で結合したDoc | §2.2(C) 親eventのクラス継承 | 独立分類なし。INDIVIDUAL eventに従属し source_ref（doc_ptr）として保持 |

### 4.2 KPI分母（個別相談件数）への効き方

- **分母の定義**: KPI分母（個別相談件数）＝ クラス `INDIVIDUAL` かつ「自動INDIVIDUAL帯（人手承認済を含む）」の consultation_key（event_id）件数。
- **効き方**:
  - SIG-2/3/4 の負シグナル除去で、型D（会議議題掲示）・型E（公益）・Gmail MLノイズが**分母から落ち**、分母の純度（個別案件率）が上がる。
  - W-210 §2 で「件数ベースの分母は明確に個別案件と判別できたものに限定し、判別不能分は別計上」とした方針を、本ルールが**灰色帯=別計上（HITLキュー）**として構造化する。捏造で埋めない。
  - Gmail INDIVIDUAL候補は背骨ID不在（W-210 §4）で相談eventと確定突合できないため、**Calendar側のINDIVIDUAL（event_id）を分母の一次母集団**とし、Gmailは流入観測の補助に留める。
- **KPI-P1-08（イベント未紐付け率）への波及**: 分母を INDIVIDUAL に絞ることで、源内 event_id↔doc_id 紐付け率（W-210 §4＝約71%）の母集団が清浄化され、未紐付け率の意味が明確になる。
