# カバレッジ・ギャップ分析 ＋ Seed辞書（A＋B）

- 作成日: 2026-06-05
- 用途: 久保さんが仕組みに**取り込める実データ**（突合マスタ・Seed辞書）と、**紐付け漏れの客観可視化**。
- 出力データ: [`/data`](../data/) 配下のCSV4点（本書はその解説とギャップ分析）。

> **方法と前提（先に明記）**
> - 突合マスタの背骨は **Gmail 案件ラベル全1,181件**（`ALO/<SF ID>_依頼者＿事件名`）を構造化したもの。**唯一の全件データ**。
> - Box 側は **現用の案件フォルダ（浅井・武市の「受任案件」＋「相談」、計141）** を取得して突合（宮澤先生は2026-04参画でフォルダ未作成、既済・裁判外・顧問は対象外）。
> - 突合は **依頼者名の正規化**（法人格・敬称・括弧書き・空白を除去）による**概算照合**。**最終確定は目視＋SF突合が必要**（Salesforce本体へは未アクセス）。
> - つまり本書の数値は「現状の客観スナップショット」であり、確定値ではない。**仕組み投入前の地ならし**として使う。

## 4. 分析で見つかった実データの知見（仕組みに効く発見）

1. **事務所は既に手作業のトリアージ・ラベルを運用していた** ― `案件外` / `要確認` / `誤分類` のラベルを検出。これは §04 の「非案件確定／グレー→人レビュー／誤紐付け訂正」が**既に現場運用されている実証**。仕組みはこの3状態を踏襲すればよい。
2. **「紹介元」は独立した重要カテゴリ** ― 法テラス・京都弁護士会・遺言相続センター・駅前夜間相談・欠陥住宅京都ネット が相談ラベルに頻出。**非案件ではなく「相談の入口」**。紹介元は当事者・分野の推定に効く別シグナル（特に武市Tの相談で顕著）。
3. **同一依頼者・複数案件が実在** ― 田中外士夫商店(3件)・龍村美術織物(2件)・福永孝文(受任+相談)。§02-3-3「依頼者一致だけで確定しない」の生きた根拠。`dict_attorney_prior.csv` に件数付きで収録。
4. **括弧書きが別名の宝庫** ― 旧称(旧サーモネット)・グループ(HIRAKU=旧はたご・ASUMO・コーチ)・当事者区分(個人/法人)・事件別名(長野地裁・GMO・三井案件) を自動抽出。別名辞書の良い出発点（要キュレーション）。

## 5. 辞書の使い方・育て方（手戻り防止）

- **これらは"Seed（種）"**。完璧ではなく、**人手キュレーション前提**。特に `dict_aliases.csv` は紹介元・裁判所名・事件別名が混在するので、**kind列で分類しながら精査**する。
- §03-5-7 / §04-7 の**フィードバック経路**に接続: 人が確定・訂正するたびに各辞書へ追記 → カバレッジが上がり、§3の漏れ候補が縮む。
- 次の精度向上ステップ（任意）: 別名辞書を当てて§3の26件を再判定／Salesforceと突合してSF側の正準名を確定／宮澤T・既済・裁判外・顧問フォルダを取り込み対象範囲を拡大。

## 6. まとめ

- **突合マスタ＋3辞書**で、設計（02〜05）が**実データの種**になった。久保さんはこれを取り込んで初期精度を底上げできる。
- 客観スナップショットとして、**現用案件フォルダの約18%（26件）がメール紐付け漏れ候補**として可視化された（要目視＋別名辞書適用）。
- 現場が既に運用する `案件外/要確認/誤分類` ラベルは、§04トリアージ設計の妥当性を裏づける。
