## 限界（本番ではない理由）
1. **突合が書名一致のみ** ＝ 設計が禁じる「タイトルのみ一致」レベル。**同定の確認ではなく積層ロジックの動作確認**。年差34・出版社差40には**別の本が同名で当たった偽ペア**が混じる（＝だからfingerprint＋TOC突合が要る、を逆に示す）。
2. **2ソースのみ**（蔵書＋弁コム）。ライオンボルト／リーガルライブラリー／NDL全量・ベンダーgenreはMac側。
3. biblio の弁コム raw は識別子＋TOCのみ＝**genre/abstract/page_count は biblio から取れず**、本proxyでは未検証（本番はcorpus層から供給）。
4. provenance_family 畳み・multi分類の合議は、ソースが2つ＆分類が蔵書単独のため**本proxyでは発火せず**（本番501で要確認）。

## 結論
**設計v0.2の中核ロジック（観測→triage→採用→接地）は実データ上で破綻せず、積層効果・低disputed・接地100%が確認できた。** ただし上記限界ゆえ、**正規の判定は Mac側の本番501 dry-run（WO投函済）を待つ**。本proxyはその妥当性に対する前向きな先行証拠。
