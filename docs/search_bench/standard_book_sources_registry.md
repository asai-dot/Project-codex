## 性質の分散（重み設計の根拠）
書店横断(1) / 弁護士媒体推薦(2,4,5,7) / 実務家キュレーション(3) / 定期刊バックナンバー(6)。
- 古典定番（江頭・菅野/水町・契約書作成の実務と書式 等）は**多系統一致**→最高重み。
- 新刊・先端本（サイバー等）は5など少数系統→別タグ（recency_signal）。
- 媒体系(2,4,5)は企業法務寄り＝**個人法務側は至誠堂(個人法務系＋家事/相続/離婚/刑事の分野別)が主軸**で補正。

## 重ね合わせ設計（号令後に実行）
1. 2〜7 を至誠堂と同パイプラインで取得（書名/著者/ISBN＋ソース内の分野/対象タグ）。
2. ISBN13 正規化→ `biblio.bib_records`(asai-bookshelf) と突合。
3. 蔵書フラグ候補に **endorsement: {source, source_type, field, recency}** を多対多で付与し、`endorsement_count` と `source_type多様性` を重みに。
4. これは Stage2 gold「事件類型×重み付き推薦本」の外部シード層。最終確定は弁護士の文献評価で上乗せ。
