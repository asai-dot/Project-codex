# 設計転換: 書式だけでなく「解説・法令・契約類型」から構造を抽出する（owner仮説の採用）

date: 2026-06-11 / owner仮説: **書式だけからの構造化は悪手（答えのないブラインドテスト）**。
書式は「結論」であって、*何を載せるべきか／載せると何が起きるか／なぜその構造か* は書式に書いていない。
それらは **解説・法令・契約類型・条項例の解説・書式の書き方** という「理由を語るテキスト」に書いてある＝**answer key**。
そこから構造を抽出して書式を議論すべき。**読み比べはLLMが既にやっている＝法律家の書式議論としては凡庸。**

## 自己批判: 私のL3(書式だけ)は何が弱かったか
- `{function:"payment", attributes:{遅延損害金率:14.6}}` のような**書式テキストの言い換え**にすぎない。
- 機能タグも私が付けた＝**抽出者＝判定者**の自己採点（ブラインド）。
- 「2つの支払条項が違う」を出せても、**なぜ14.6%か／載せないと何が起きるか／根拠法**は書式に無いので出せない。
- ＝世のLLMの「大量読み比べ」と同じで、法律家の議論には軽い。**owner指摘は正しい。**

## 解説接地の抽出（実例: 再委託）— `poc_grounded/saiitaku.knowledge.json`
同じ業務委託本の**第3章「解説」**(pp.75-78)を読んで、書式でなく**設計知識**を抽出した:
- **なぜ(rationale)**: 当事者は委託者と受託者のみ→再委託は受託者の責任で行われる→無限定は委託者に不利→**原則禁止＋事前承諾**が一般的。
- **何を載せるべきか(must_address)**: 禁止範囲(全部/主たる部分/一部)・事前承諾の要否・**再委託先の行為への責任の所在**・報酬の支払方法。
- **載せると何が起きるか(design_options→effect)**:
  - 責任：①全責任(委託者標準) ②「選定・監督についてのみ責任」=**受託者の責任限定**。再委託先が委託者指定なら説得力増。
  - **法令根拠**: 旧民法105条1項(復代理人の選任監督責任)は**現行民法で削除→限定説の根拠が弱化**(脚注14・一問一答27頁)。
- **書式リンク**: 〈再委託原則禁止の条項例〉は、この設計知識の**一インスタンス**にすぎない。

→ これは私の「書式だけL3」では原理的に出ない。**答え(解説テキスト)に接地しているのでブラインドでない**。
  そして**我々の蔵書は「解説＋書式」のペア**であり、toc_nodesが解説節と書式を頁で繋げる＝この抽出を全コーパスで回せる。

## 再設計の骨子（構造の出どころを変える）
| | 旧(私のL3) | 新(解説接地) |
|---|---|---|
| 構造の源 | 書式テキスト | **解説/法令/契約類型/条項例解説/書式の書き方** |
| 単位 | 条項→機能タグ(私の推測) | **条項機能の設計知識**(rationale/must_address/options→effect/legal_basis、**引用付き**) |
| 書式の位置 | 真実の所在 | **一インスタンス**(設計知識へリンク) |
| 検証 | 抽出者=判定者(ブラインド) | **解説テキストが正解**（引用で追跡可能） |
| 議論 | 書式の読み比べ(=LLM凡庸) | **なぜ/効果/根拠法に接地した法律家の議論** |

## 我々が既に持っている材料（この設計が現実的な理由）
- **解説リッチな蔵書**: 「作成のポイント」「契約書作成の実務と書式(条項例と解説)」「○○の作り方」等＝**条項ごとに解説**。
- **toc_nodes**: 解説節↔条項例を頁で接続できる（同一書内）。
- **法令テキスト**(e-Gov等)・契約類型の体系書(LION BOLT/蔵書)。
- vision OCRで**解説頁**を読める（書式頁だけでなく）。

## 次（本設計）
1. **抽出単位を「条項機能の設計知識」に変更**（schema `clause_design_knowledge`）。源＝解説/法令、引用必須。
2. **接地パイプライン**: toc_nodesで「解説節」と「条項例」を対にし、解説をOCL→設計知識を抽出→書式にリンク。
3. **法令・判例・一問一答へのリンク**（再委託の民法105条のように）。
4. コーパス議論は**設計知識ベース**（「再委託の責任の取り方の選択肢と効果」を蔵書横断で）。読み比べでなく接地議論。
5. 精度検証は**引用追跡**で可能（抽出が解説テキストに根ざすか）＝ブラインド脱却。
