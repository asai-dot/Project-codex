# 評価 — 契約書式実務全書 第1〜3巻（ぎょうせい）を tmplstruct 材料にするか

**結論（先に）**: 素材としては**最高クラスの純・契約書式集**。ただし既存式とは別の「**新規ハーベスト**」になり、
本文が超大容量ゆえ扱いに工夫が要る。**目次OCRで骨格化までは今すぐこの環境で可能**、本文は別経路。

## 実データ確認（2026-06-11）
| 項目 | 結果 |
|---|---|
| 所蔵 | **全3巻 所蔵**（asai-bookshelf）。さらに**新版(9784324107508/515/522)も所蔵**＝新旧2版そろい |
| 出典本か | **否**（リーガルライブラリー外＝bencom無し）。既存6,976式に未リンク＝**新規式の素材** |
| 詳細目次 | DB `bib_toc`は**無し(0行)**。ただし Box `app/data/toc/isbn_9784324096970.json` 等に **codex_ocr の TOC が既存**（648項目）。但し**「総目次」レベル＝2階層・頁なし(page_start全None)・生OCRノイズ有**で、**式単位の細目次・頁は未整備**。→ 式・頁は `02_目次`(細目次)のOCRで補完が要る（実証済） |
| 自炊4点 | **3点（索引なし）**。書式集なので目次が主で影響小 |
| ファイルサイズ | `01_全体`= 第1巻 **996MB** / 第2巻 **1.1GB** / 第3巻 **954MB** → **全てBox変換上限(約642MB)超**。`02_目次`= 11〜17MB（変換可） |

## 実証
- **02_目次のOCRは成功**（第1巻 目次p3＝「第Ⅰ編 契約総論」を、編→項[1][2][3]→細目＋執筆者＋頁 で抽出）。
  → bib_tocが無い本でも、**目次画像OCRで頁付き骨格を生成できる**。
- 一方 `01_全体`(約1GB) は **この環境内OCR不可**（Box変換せず）。本文の書式OCRは：
  - **Mac側レンダリング**（`pdftoppm` → 画像 → OCR）、または
  - **OCR用に分割/300dpi派生PDF**（≤約250〜365MB）を作って併置 → この環境内OCR可。

## 進め方の判断
- **今すぐ価値が出る**: 全3巻の `02_目次` をこの環境でOCR → **式（契約書式）リスト＋頁の骨格**を作る（新規テンプレ候補の一覧化）。
- **本文の構造化**: 01_全体が大容量のため、(a)Mac側 or (b)分割/300dpi派生 のどちらかを用意してから。
- **位置づけ**: 既存tmplstruct(リーガルライブラリー由来6,976式)とは**別レーン**＝「自前所蔵の書式集からの新規ハーベスト」。
  既存式の精度改善（再自炊→ポイントOCR）とは独立に、**新規式の供給源**として価値が高い。

## 推奨アクション
1. （即）全3巻 `02_目次`(細目次) をOCR → 既存の総目次JSON(`app/data/toc/isbn_*.json`)に**式名・頁・執筆者を補完** → `keiyaku_zensho_skeleton.json`（編/契約類型/式名/頁/執筆者）を作成。
2. （要準備）本文OCRは Mac側 or 分割派生PDF を用意して着手。
3. 新旧2版あるので、**正規化は新版**を基準にすると将来の参照に強い。
