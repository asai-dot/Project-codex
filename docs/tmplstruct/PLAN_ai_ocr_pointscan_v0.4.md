# tmplstruct 構造化 v0.4 計画 — 詳細目次(bib_toc)＋書式ページ ポイントOCR

**前提認識（2026-06-11 確定）**: リーガルライブラリーの全文OCRは精度が頭打ち。式の構造化は
「① 構造の骨格は既存の**詳細目次**を使う／② **書式ページだけ**を最新ビジョンLLMで**ポイントOCR**して式の中身を撮る」
の2トラックで作る。目次画像のOCRし直しはしない。

---

## 1. 方針（2トラック）
| | 何を | どこから | OCR要否 |
|---|---|---|---|
| **① 骨格** | 章節〜式名の階層・頁・順序 | `biblio.bib_toc`（既存・詳細目次） | **不要** |
| **② 中身** | 各書式の内部構造（条項・空欄・署名欄・別記/別紙） | 書式ページの画像のみ | **ポイントOCR（ビジョン）** |

→ 全文を舐めない。**目次で式の頁を特定 → その頁だけ撮る**。コストも誤差も最小。

## 2. 確定済みの資産（実確認）
- `biblio.bib_toc`：**552,544行 / 3,802冊（bencom側）**、列= `bib_id, ordinal, level, page, text`。
  例『会社議事録の作り方〈第3版〉』= **456行・4階層・頁付き**（式名と頁が揃っている）。
- `biblio.bib_records`（source=`bencom-library`）に各出典本の bencom bib_id・`source_url`（content_idハッシュ）。
- 出典本175冊／式6,976（`docs/tmplstruct/sourcebook_match/source_books_175.csv`）。うち**89冊を現物所蔵**（DB確認済）。
- 自炊600dpi画像は一部のみ（`sourcebook_jisui_verified.csv`）。**本丸の大型書式本は完全自炊が未整備**。

## 3. パイプライン（式抽出に最適化）
```
[S1] 式インデックス確定
     templates(6,976) を 出典本 bencom bib_id に紐付け
[S2] 頁解決（OCR不要）
     各式 → bib_toc の該当行(text≈式名, level深) を引き当て、page を取得
     式の頁範囲 = この式のpage 〜 次の同階層/上位エントリのpage-1
[S3] 書式ページ抽出
     範囲の頁だけを 画像ソースから取り出す（※画像ソースは §4 で決定）
[S4] ポイントOCR（ビジョンLLM）
     書式ページ画像 → 式の内部構造JSON（条項/空欄[___]/選択肢/署名欄/別紙/脚注）
[S5] 構造化出力
     tmplstruct スキーマへ（式境界・見出し階層は①、中身は④）
     → 反映先: biblio.bib_toc（骨格）＋ tmplstruct 構造テーブル/JSON
```

## 4. ②の画像ソース 【owner決定済 2026-06-11】
**基準＝うちの自炊600dpi画像**（高画質で将来精度が出る）。bencomページ画像は使わない。
**本丸（会社議事録/会社法務書式集第3版/企業労働法書式編）は再自炊で画像を揃える**前提。
- 既に自炊4点がある本（契約解消・契約書作成の実務と書式・業務委託・情状弁護アドバンス等）は即OCR可。
- 本丸・未自炊本は `スキャンルール.txt` 準拠（600dpi/4点）で**再自炊→OCR**。

> 実行上の制約: リモート環境の Box `get_preview_page` は空(0byte)を返すため、**画像取得＋vision呼び出しはMac/ワーカー側**で行う（`tools/pointocr_pilot.py`）。

## 5. ポイントOCR 仕様（②）
- モデル: Claude（vision）。入力＝書式ページ画像（1式＝数頁）。
- 出力JSON（式1件）:
```json
{
  "form_title": "株主総会議事録（定時・普通決議）",
  "source": {"bib_id":"NOBN_...","page_from":120,"page_to":123},
  "blocks": [
    {"type":"heading","text":"株主総会議事録"},
    {"type":"clause","no":"第1","text":"...","blanks":["開催日","場所"]},
    {"type":"signature","roles":["議長","出席取締役"]},
    {"type":"attachment","ref":"別紙1"}
  ],
  "blanks_total": 7,
  "notes": "脚注・記載上の注意の有無"
}
```
- プロンプト固定方針: 「**書式の体裁を保ったまま**、条項番号・空欄(___)・選択肢(□)・署名欄・別紙参照を構造化。解説本文は拾わない」。
- 検算: `blanks_total`・条項数を人手スポット確認（式名取りこぼしのみ要チェック）。

## 6. フェーズ
- **P0 パイロット（1冊）**: 自炊画像が既にある中規模式本でS2→S5を一気通貫。
  候補: `契約解消の法律実務`(14式・4点完備) か `情状弁護アドバンス`(111式・（未）4点あり)。
  → bib_toc頁解決の的中率＋ポイントOCRの構造精度を実測。
- **P1 本丸3冊**: 画像ソース確定後（§4）、会社議事録/会社法務書式集第3版/企業労働法書式編。
- **P2 横展開**: 所蔵×式数の多い順に展開（所蔵89冊=3,762式が射程）。

## 7. コスト/制約
- 全文でなく**書式頁のみ**なので、OCR対象は式6,976×数頁に限定（全頁の数分の一）。
- 600dpi `01_全体` は巨大（数百MB）。**頁範囲だけ抽出**してから画像化（S3）。
- 実行はMac/ワーカー側（画像化＋vision呼び出し）。本リポジトリでは設計・スキーマ・プロンプトを確定。
