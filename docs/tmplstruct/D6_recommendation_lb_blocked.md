## 結論：両方とも **採用A**（カード推奨に同意・上記で裏取り済み）＋ 系統化を1点提案

### D-6-a 独占禁止法〔第4版〕(白石・有斐閣, YHK0001663) → **構造から確定で直す（review先送りにしない）**
- 所蔵: **あり**（`alo:book:manual:title_6238` / 9784641243668, asai-bookshelf）。ただし**自炊フォルダに無し＝未スキャン**（自炊済は条解/論点体系の独禁法本で、白石本は別）。→ 現物画像での目視は今は不可。
- だが**現物を見るまでもなく確定可能**。LION BOLTの実データ構造：
  `序章`(ord=0・最初) → `第1部 違反要件総論`(ord=90) → `第2部`(279) → `第3部`。
  序章は**どの部より前に出る最上位の前付け章**であり、`第X部`(level0)の子であり得ない。LION BOLTの level 揺れ（序章をlv1に誤付与）が原因。
- **推奨（更新）**: review昇格に留めず、**`序章`を最上位ノード（level0／親なしroot）へ確定補正**する。
  - これで `first_row_depth_gt_1` も構造不整合も解消。残り410ノードは元から正常。
  - 補正は**levelの補正のみ**でノード発明ゼロ（D-6-bの level正規化と同種の処理＝framework整合）。
  - 白石本の実体（序章＋3部構成）とも一致。**review無期限放置より確実に良い**。
- もし序章の厳密な階層（部と同列か章扱いか）を**完全に視認**したい場合のみ、**所蔵の現物を1冊スキャン**して目次を確認すればよい（任意・必須ではない）。本ブロック解消には上記の確定補正で十分。

### D-6-b 大コンメンタール刑訴〔三版〕第6巻(青林書院, SRN0000263) → **level正規化(A)に賛成**
- これは典型的な**多巻本の1巻**（10巻組の第6巻＝第282〜307条）。level 0-2(編・章・節)は**全巻共通の上位構造**で、本巻のTOCではない。
- 巻単体では「第X条」が事実上のルート。**`level - content_min_level + 1` で正規化＋末尾バックマター除外**で `first_row_depth_gt_1` は解消し、正当な逐条TOCになる。
- セット↔巻の関係は DD-LITID 親子レーンで処理済み（v0.3 §5）→ TOC側で合成親を作らない方針とも整合。**異論なし**。

## 追加提案（再発防止・系統化）
カードの実装案は2冊を**ハードコード**（`D6_NORMALIZE_BOOKS={"LB_SRN0000263":...}`）。
だが「**多巻コンメの巻フラグメント**」は今後も必ず再発する（大コンメンタール/注釈系は巻数が多い）。
- 推奨: `first_row_depth_gt_1` かつ「末尾に孤立 level0 バックマター」かつ「content開始levelが一定」のシグネチャを
  **一般則として検出→巻ルート正規化（review flag付）**にする。
- 効果: 巻フラグメントが出るたびに owner決定カードを起こす運用コストを無くせる。**個別ハードコードは初回のみ、以降は規則で自動**。
- 安全性: 正規化は「level の基底シフト＋バックマター隔離」だけで**ノード発明ゼロ**（gate_no_node_invention維持）。review flagで監査可能。
