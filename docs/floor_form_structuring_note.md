# 書式構造化のヒント — 法定床の合成と整合の錨

owner 着想(2026-06-25): 「登記事項とか法定記載事項とかと組み合わせるとさらに精度上がりそう。
書式の構造化のヒントの一つやな。」

## 命題

1つの書式の slot構造 ＝ **その手続で作る各書面が背負う、複数の法定床の合成**。

```
募集株式発行(手続)
 ├ 議事録        → 床: 会社法199①(募集事項 各号)
 ├ 登記申請      → 床: 会社法911③/変更登記の登記事項 + 商業登記法
 └ (定款変更時)  → 床: 会社法27(定款の絶対的記載事項)
```

- **書式の必須 slot = Σ(適用される法定床)**。条文各号から床を取れる(`floor_check`/`floor_sweep`)ので、
  書式の構造プロファイル(⑦)を **条文から逆算** できる。
- **複数床に共通して現れる項目 = 書面間の整合チェックの錨**。同じ値が議事録と登記で食い違えば誤り。

## 計算で裏付け(`scripts/floor_link.py`)

定款27 × 募集株式199① × 設立登記911③ の床を横断し、共通語(整合の錨)を機械抽出:

```
【資本金】       募集株式199①五号 ↔ 設立登記911③五号   ← 議事録の資本金額 = 登記の資本金額
【種類株式…】    募集株式199①一号 ↔ 設立登記911③       ← 株式の種類
【名称及び…】    定款27五号        ↔ 設立登記911③       ← 発起人/役員等の名称
【氏名又は…】    定款27五号        ↔ 設立登記911③       ← 氏名
```

⇒ 「資本金」は **議事録(199①五)で決定 → 登記(911③五)へ転記** される値。両書面で一致すべき
**整合の錨**。これを書式の slot として固定し、書面間で値クロスチェックすれば精度が上がる。

## 何に効くか(⑦書式構造化への接続)

- **slot 導出**: 書式の structure_profile を「どの法定床を背負うか」から半自動生成できる。
  手書きの slot 定義より漏れにくい(条文が正本)。
- **整合ルール**: 共通の錨(資本金・発行可能株式総数・商号・本店 等)に、書面間の値一致ルールを張る。
- **抜け漏れ × 整合 の二段**: `floor_check`(各書面が床を満たすか) + floor_link(書面間で値が整合するか)。

## 次段(やるなら)

1. `procedure → applicable_floors[]` の束(bundle)を定義(例 募集株式発行 = {議事録:199①, 登記:変更登記}).
2. 書面セットを一括で「各床充足 + 錨の値一致」チェック。
3. 錨語の正規化(現状 漢字連続のみ抽出 = 「及び/又は」で切れる。語境界を整えると錨がより綺麗に出る)。
