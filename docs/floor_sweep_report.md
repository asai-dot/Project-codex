## 読み方と有効性の判断

- **構造抽出: 10/10手続で成功**。条文1本(会社法XML)から104の法定記載事項を機械抽出できた。
  他法令(商業登記法・不登法等)も e-Gov から1回取得すれば同様に横展開可能。
- **検出機構: recall 98%**。各号を1つ落とすと、ほぼ必ず「欠落」として撃てる。
  漏れた2件(676社債/911登記)は、ある号の名称が別の号の名称の部分文字列で、落としても別号経由で
  在りと判定された取りこぼし(入れ子名称)。固有名で照合すれば解消する既知の改善点。
- **実prose照合: 77%の号が alias 整備待ち**。条文名称は括弧書き・条件節で長く、実務書面の口語
  (例「募集株式の**種類及び**数」)と素では当たらない。**ここが production 化の主コスト**。
  募集株式199①で実証済(1号 alias を足して文例33型が ✅ になった)。

## 結論(有効性)

- **骨格は広く効く**: 抽出+検出は会社法10手続で機構的に成立(104各号/検出98%)。
- **実戦投入の律速は alias curation**: 法定記載事項の口語別名を整える作業(77%の号)。
  高頻度手続から人手で整える / 条文名称からの別名候補を半自動生成して owner レビュー、のいずれか。
- 実prose での合否は `pipeline/floor/examples/`(文例33型=合格 / 払込期日もれ=四号検出)で確認済。
