## 背景（v0.2 の残課題）

v0.2 実行後、`法学` が canonical 最大の **46,279件**。中身を `掲載誌等` 先頭で割ると
別誌の寄せ集めと判明（＝広すぎる検索語「法学」で D1 を全部入りで落としたフォルダ）:

| 隠れていた実誌 | 件数 |
|---|---|
| 法学セミナー | 8,535 |
| 月刊法学教室 | 5,959 |
| 法学研究（慶応） | 1,833 |
| **法学協会雑誌（評釈4位）** | **1,297** |
| 立命館法学 / 法学新報（中央）/ 阪大法学 … | 各〜1,000 |

→ データは在るが誌名が「法学」に潰れ、**評釈4位の法学協会雑誌が独立カウントされず埋もれていた**。
再取得は不要。**ラベラー側で `掲載誌等` を使い割り直す**のが正解。

## 方針

- 各 canonical フォルダで、配下レコードの `掲載誌等` 先頭（=記事ごとの実誌名）の**分布**を見る。
- `件数≥300 かつ 最頻誌シェア<0.5 かつ 実質4誌以上に散る` を **catch-all** と判定し、
  そのフォルダのレコードだけ `掲載誌等` 先頭で誌名を割り直す。
- **単一誌の表記ゆれ（最頻誌が支配的）は分割しない** ＝ 金融法務事情↔旬刊金融法務事情の誤分裂、
  銀行法務21→銀行法務 の数字破壊を防ぐ。純粋フォルダ（ジュリスト等）も **v0.2 のまま不変**。
- catch-all で救出した誌は、独立フォルダで再取得済みの同名誌と canonical 一致で **自動統合**。
- 非破壊・冪等: `<build>/labeled_v0.2.1/` に新規出力、件数(282,761)不変。

> 注: 当初の判定は「フォルダ名との一致率」だったが、同一誌が別表記（旬刊金融法務事情等）の
> フォルダを誤って catch-all 判定し1誌を分裂させたため、**最頻誌シェア（多様性）ベースに変更**した。

`掲載誌等` 先頭抽出: NFKC → 先頭の括弧/引用記号除去 → 最初の数字(巻号)で切る →
大学名などの括弧修飾は残す（誌の区別に効く）。

## 検証（合成データ）

catch-all「法学」(別誌5種×60=300, 最頻0.2)を分割し各誌に展開、単一誌の表記ゆれ「金融法務事情」
(旬刊0.63)と数字名「銀行法務21」(~1.0)は分割せず保持、純粋「ジュリスト」(400)不変、
独立「法学協会雑誌」(50)＋catch-all由来(60)=110 に統合、`?`=0 を確認。
