## まず守る3原則

1. **スコープを守る** — 指示されたファイル・範囲だけ触る。「ついで」の変更をしない。
2. **小さく確実に** — 差分は最小単位。1コミット=1つの意味。テストで裏を取る。
3. **迷ったら止めて訊く** — 仕様が曖昧・矛盾・スキーマ変更が必要なら、進めずにヘッドへ確認を上げる。
   「わからない」と言えるのが真面目さ。

## 作業の進め方

1. 受け取ったタスクの **受け入れ基準(DoD)** とサンプル入出力を確認する（無ければ訊く）。
2. 関連ファイル／既存コードの流儀を読み、**周囲に合わせて**書く（命名・構造・コメント密度）。
3. 実装する。可能な限り**テスト or 検証スクリプト**を同時に用意する。
4. 自己チェック（下記）を回す。
5. 小さくコミット → ブランチに push → ドラフトPR。ヘッドと GPT監査のレビューを待つ。

## データを触るときの絶対ルール

成果物は JSONL（1行=1定義レコード）。スキーマは固定で、無断変更禁止（ヘッド承認制）。

```json
{"term": "後見人", "definition": "未成年後見人及び成年後見人をいう。", "law_id": "129AC0000000089", "law_name": "民法", "article": "10", "item": null, "uri": "egov:129AC0000000089:art:10", "scheme": "jp_statutory_definition", "authority_rank": 100, "source": "egov", "definition_type": "paren_definition", "confidence": "high"}
```

必須キー: `term, definition, law_id, law_name, article, item, uri, scheme, authority_rank, source, definition_type, confidence`

- `definition` は **原文に忠実**。要約・言い換え・創作は禁止（**ハルシネーション厳禁**）。
- 根拠 `uri` は必ず埋める。たどれない定義行は出さない。
- 確信が持てない行は捨てるか `confidence` を下げる。価値序列は **正確さ ＞ 網羅性 ＞ 速度**。
- すべての行が**有効なJSON**で、上記キーを過不足なく持つこと。
